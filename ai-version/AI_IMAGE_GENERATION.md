# AI Image Generation Guidelines

This document outlines the guidelines and process for generating card images using AI.

## Card Image Requirements

### Critical Rules

**IMPORTANT: Card images generated should NEVER contain text**

Card images are meant to display visual artwork only. Any text (card names, titles, descriptions, etc.) should be rendered separately by the game UI, not baked into the image.

## The `aiPrompt` Field

Each card in `src/constants/CARDS.ts` has an `aiPrompt` field:

```typescript
aiPrompt: string;
```

### Purpose

- **Empty string (`''`)**: The image was NOT generated by AI. It's a manually created or sourced image.
- **Non-empty string**: Contains the exact prompt used to generate the card image via AI.

### Benefits

1. **Traceability**: Know exactly how each AI-generated image was created
2. **Consistency**: Regenerate or create variations using the same prompt structure
3. **Documentation**: Future card creators can reference successful prompts
4. **Versioning**: If images need to be regenerated, the original prompt is preserved

## Using the gemini-image-generator Skill

### Basic Usage

```bash
# From the ai-version directory
./.claude/skills/gemini-image-generator/scripts/generate.py \
  --prompt "Your detailed image description" \
  --output static/images/creature/[element]/[card-name].jpg \
  --size 2K
```

### Prompt Guidelines

When creating prompts for card images:

1. **Be specific about the subject**: Character type, species, role
2. **Describe visual elements**: Armor, weapons, magic effects
3. **Set the atmosphere**: Background, lighting, weather
4. **Specify art style**: "epic fantasy trading card game art style, highly detailed digital painting"
5. **Emphasize NO TEXT**: While not explicitly stated in prompt, ensure generated images don't include text overlays

### Example Prompt Structure

```
A [adjective] [creature type] [doing action], [physical description],
[clothing/armor details], [magical effects], [background description],
[art style], [lighting/atmosphere]
```

### Example

```
A fierce humanoid lightning warrior, muscular athletic build,
wielding crackling blue-white electricity in his hands, dark storm clouds
in dramatic background, electric blue and yellow lightning energy swirling
around his body, wearing medieval fantasy armor with lightning bolt motifs
and gold trim, intense determined expression, dynamic action pose mid-strike,
epic fantasy trading card game art style, highly detailed digital painting,
dramatic cinematic lighting
```

## Workflow

1. **Design the card** in `CARDS.ts` with stats, keywords, etc.
2. **Write the AI prompt** considering the guidelines above
3. **Generate the image** using the gemini-image-generator skill
4. **Save the image** to `static/images/creature/[element]/[card-name].jpg`
5. **Store the prompt** in the card's `aiPrompt` field
6. **Verify** the image has no text overlays

## Image Specifications

- **Format**: JPEG (.jpg)
- **Size**: 2K (recommended for card images)
- **Location**: `static/images/creature/[element]/[card-name].jpg`
- **Naming**: Use lowercase with hyphens (e.g., `lightning-warrior.jpg`)

## Model Information

- **Current Model**: `gemini-3-pro-image-preview`
- **API**: Google Gemini
- **Configuration**: Set `GEMINI_API_KEY` environment variable in `.env.local`
